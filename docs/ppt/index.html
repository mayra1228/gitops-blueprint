02 / 10
PAIN POINTS · 四大痛点

当前痛点

01 · SCATTERED
IaC 配置分散
Terraform / YAML 散落在多个仓库，无统一产品化视图。缺乏资源拓扑和依赖关系可视化，运维认知负荷高。
02 · BROKEN FLOW
变更流程断裂
Plan / Apply 缺少审批门禁，靠人工 Review 把控质量。无审计追踪，事后难以追溯变更链路。
03 · NO GUARDRAIL
执行缺乏安全边界
任何人可在任何集群/环境执行 apply，无策略门禁、无集群级熔断，生产环境面临严重风险。
04 · NO AI ASSIST
AI 辅助空白
安全审计靠人眼，缺少自动化检查。执行失败诊断依赖个人经验，平均排障耗时长。

Security & Privacy · Dual Guard + Audit
06 / 10
EXECUTION GUARD + DATA PRIVACY

安全边界 & 数据隐私

EXECUTION SECURITY

🛡️ 双重守卫

Guard 1 · Platform Backend

adapter 层强制校验集群名称，非 kind-gitops-sandbox → 拒绝

Guard 2 · GitHub Actions

Workflow 脚本二次校验，独立于平台的安全断言

环境策略矩阵

dev/sandbox → auto-approve
staging → auto-approve + manual-apply
prod → 2 人审批 + manual-apply

当前禁止

🚫 生产 apply · 🚫 Jenkins trigger · 🚫 自动 merge · 🚫 非测试仓库

DATA PRIVACY

🔒 数据隐私

Prompt 脱敏 · 始终执行

password, secret, token, access_key, secret_key, private_key, credential → ***REDACTED***

data_policy 强制

no_external → 阻断非 localhost 调用
allow_external → 允许但仍脱敏

Anti-Sync-Storm

每对象限速 5/小时，AI→变更→AI 循环检测，AI 输出为惰性数据不可调用变更

全链路审计

prompt_hash · model · token_count · latency · response_hash → audit trail
